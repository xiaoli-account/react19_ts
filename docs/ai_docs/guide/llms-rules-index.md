# llms.full.txt 规则编号索引

> 本文档提供 llms.full.txt 中所有规则的快速索引和说明

## 📋 规则编号体系

### 编号前缀说明
- **FR** (Forced Rules) - 强制规则：必须严格遵守
- **ER** (Execution Rules) - 执行规则：开发过程中必须遵守
- **AR** (Advisory Rules) - 建议规则：可根据实际情况灵活调整

### 重要性标记
- ⭐⭐⭐ = 核心规则，必须优先执行

---

## 🔴 强制规则 (Forced Rules)

### FR-01 自检机制 ⭐⭐⭐
**作用**: 代码生成前的自检清单
**关键点**: 
- 判断是否为独立功能
- 预估代码行数
- 确定存放目录

**判断逻辑**:
- 独立功能 OR 代码 ≥ 500 行 → `src_ai/`
- 非独立功能 AND 代码 < 500 行 → `src/`

---

### FR-02 代码安全性
**作用**: 确保生成的代码不破坏项目
**关键点**: 不影响项目运行、不破坏项目结构、不破坏项目功能

---

### FR-03 代码行数预估
**作用**: 生成代码前预估总行数
**关键点**: 提前判断代码规模，决定存放位置

---

### FR-04 目录结构理解
**作用**: 理解 src/ 和 src_ai/ 的关系
**关键点**: 
- `src/` - 手动编写的代码
- `src_ai/` - AI 生成的代码
- 两者并列，共同构成项目

---

### FR-05 目录作用定位
**作用**: 明确 src_ai/ 的定位
**关键点**: src_ai/ 是正式项目的一部分，为 src/ 提供辅助

---

### FR-06 代码存放规则 ⭐⭐⭐
**作用**: 决定代码存放位置的核心规则
**优先级**:
1. **独立功能规则** (最高优先级)
   - 独立功能/页面/模块/组件 → 必须放 `src_ai/`
2. **大文件规则**
   - 代码 ≥ 500 行 → 必须放 `src_ai/`
3. **小文件规则**
   - 代码 < 500 行 且 非独立功能 → 可放 `src/`

**应用案例**:
- Dashboard 首页 (652 行，独立页面) → `src_ai/pages/dashboard/`
- 修改登录页 (50 行改动) → `src/pages/login/`
- 通用工具函数 (100 行) → `src/utils/`
- 独立图表组件 (300 行) → `src_ai/components/chart/`

---

### FR-07 src_ai 目录作用
**作用**: 说明 src_ai/ 的主要作用
**关键点**: 
- 区分 AI 代码和手动代码
- 不是临时中间件

---

### FR-08 测试代码存放
**作用**: 规定测试代码的存放位置
**关键点**: 测试相关代码 → `src_ai/test/`

---

### FR-09 文档管理规则
**作用**: 规定文档的命名和存放
**关键点**:
- 文档默认使用中文命名
- AI 生成的文档 → `docs/ai_docs/{类型}/`
- 用户指定路径的文档 → 按用户指定

---

## 🟡 执行规则 (Execution Rules)

### ER-01 技术栈约束
**作用**: 规定项目使用的技术栈
**关键技术**:
- React 19.2.x + TypeScript 5.9.x + Vite 7.2.x
- Ant Design 6.1.x (禁用其他 UI 框架)
- Zustand 5.0.x (禁用 Redux/MobX)
- React Router 7.12.x
- i18next 25.7.x + react-i18next 16.5.x
- ECharts 6.x

---

### ER-02 命名规范
**作用**: 统一项目命名规范
**关键规范**:
- 目录: kebab-case
- 页面/组件入口: index.tsx
- Hooks: use-xxx.ts
- 样式: styles.scss / styles.module.scss
- SCSS 变量: $lee-xxx
- CSS 变量: --lee-xxx
- 国际化: lee-layout- (框架层)

---

### ER-03 项目结构约定
**作用**: 规定项目目录结构
**关键目录**:
- `src/components/` - 全局通用组件
- `src/pages/` - 路由级页面
- `src/hooks/` - 通用 Hooks
- `src/services/` - API 服务层
- `src/store/` - 全局状态管理
- `src/utils/` - 工具函数

---

## 🎯 快速决策树

### 我应该把代码放在哪里？

```
开始
  ↓
是否为独立功能/页面/模块/组件？
  ├─ 是 → src_ai/ (FR-06.a)
  └─ 否 ↓
       代码是否 ≥ 500 行？
         ├─ 是 → src_ai/ (FR-06.b)
         └─ 否 → src/ (FR-06.c)
```

### 我应该如何命名？

```
文件类型
  ├─ 目录 → kebab-case (ER-02)
  ├─ 页面入口 → index.tsx (ER-02)
  ├─ 组件入口 → index.tsx (ER-02)
  ├─ Hooks → use-xxx.ts (ER-02)
  ├─ 样式 → styles.scss (ER-02)
  └─ 文档 → 中文命名 (FR-09)
```

## 🔍 规则查询
