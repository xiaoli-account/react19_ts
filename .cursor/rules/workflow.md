# 开发工作流

## 创建新功能

1. **检查现有实现**: 创建新功能时，先检查是否已有类似实现
2. **遵循目录结构**: 遵循项目的目录结构和命名规范
3. **复用现有代码**: 使用项目已有的工具函数和组件
4. **代码检查**: 确保代码通过 ESLint 和 TypeScript 检查

## 代码生成流程

1. 阅读项目知识库文件
2. 确定代码存放位置（根据代码行数）
3. 遵循命名规范和代码规范
4. 使用正确的技术栈
5. 添加类型定义和注释
6. 运行代码检查

## 代码审查

- 确保代码符合 TypeScript 类型安全要求
- 确保代码通过 ESLint 检查
- 确保代码符合项目规范
- 确保代码符合命名规范

## 测试

- 确保功能正常工作
- 确保没有破坏现有功能
- 确保代码质量符合要求
