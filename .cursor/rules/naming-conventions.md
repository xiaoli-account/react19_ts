# 命名规范

## 目录命名

- 使用 **kebab-case** (短横线命名)
- 示例: `user-management`, `role-management`, `system-settings`

## 组件命名

- 使用 **PascalCase** (大驼峰命名)
- 示例: `UserCard.tsx`, `RoleTable.tsx`, `SystemSettings.tsx`

## Hooks 命名

- 使用 `use-xxx.ts` 格式
- 示例: `use-auth.ts`, `use-permission.ts`, `use-table.ts`

## 样式文件命名

- 统一命名为 `styles.scss`
- 与组件文件放在同一目录

## 工具函数命名

- 使用 **camelCase** (小驼峰命名)
- 示例: `getUserInfo`, `formatDate`, `validateEmail`

## 类型定义文件

- 统一命名为 `types.ts`
- 与组件文件放在同一目录

## 服务文件命名

- 使用 **camelCase** 或 **kebab-case**
- 示例: `userService.ts` 或 `user-service.ts`
